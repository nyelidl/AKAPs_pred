# Hard-negative benchmark — README

Biology-first hard-negative challenge set for **AKAPSpred v5.1 + ML v2**. Goal:
test whether the deployed pipeline separates true PKA-anchoring AKAP helices from
real non-AKAP amphipathic helices. Diagnostic only — nothing is retrained or
retuned.

## Quick start (after you provide verified sequences)

```
python3 hard_negative_qc.py --meta <verified_metadata.csv> --fasta hard_negative_amphipathic_set.fasta
python3 run_hard_negative_benchmark.py --fasta hard_negative_amphipathic_set.fasta \
        --meta <verified_metadata.csv> --screen ../akap_screen.py --outdir .
```

The harness expects `akap_screen.py` + `akap_pssm.json` + `akap_ml_model_v2.joblib`
on hand (the deployed v5.1 package); pass its path via `--screen` if not in the
parent directory.

## Two things to know

1. **No sequence is fabricated.** Candidates carry accessions/regions only; every
   `verification_status` must be cleared (confirm accession, isoform, region —
   several regions are approximate) before fetching.
2. **Ambiguous candidates are flagged, not used.** The DDIP-like entries
   (`DD01`, `DD02`) are marked `verify_..._AKAP_status`; QC blocks them from
   training use until their non-PKA-anchoring status is confirmed.

## Reading the result

- No danger-zone entry for a class → PSSM already filters it.
- Danger-zone entry + low ML → ML v2 adds real discrimination there.
- Danger-zone entry + high ML → priority ML v3 hard-negative class.
- Full-protein FPs in the known helix region are the strongest signal; peptide-only
  FPs are reported separately.
