คู่มือใช้งาน AKAP Domain Screener

คู่มือภาษาไทยสำหรับผู้ใช้ทั่วไป: วิธีใส่ข้อมูลโปรตีน วิธีกดรันโปรแกรม และวิธีแปลผลลัพธ์จากตาราง

เว็บโปรแกรม: https://akapspred.streamlit.app/

🧬 AKAP motif screening 🎯 RI / RII specificity ⚠️ DDIP / negative determinant flags

โปรแกรมนี้ใช้ทำอะไร?

AKAP Domain Screener ใช้สแกนลำดับโปรตีนเพื่อหา candidate AKAP motif หรือบริเวณ alpha-helix ที่อาจจับกับ PKA regulatory subunit ได้

สำคัญ: ผลลัพธ์จากโปรแกรมเป็นการคัดกรองและจัดลำดับความน่าสนใจ ไม่ใช่หลักฐานยืนยันการจับกับ PKA โดยตรง ควรตรวจสอบต่อด้วยข้อมูลโครงสร้าง การอนุรักษ์เชิงวิวัฒนาการ และการทดลอง

วิธีใช้แบบเร็ว

  1. เปิดเว็บ AKAP Domain Screener
  2. เลือกเมนู Search by protein name / UniProt ID
  3. ใส่ชื่อโปรตีนหรือ UniProt ID ทีละบรรทัด เช่น SPHKAP, Ezrin, AKAP10, O43572
  4. เลือก organism เช่น Human
  5. ติ๊ก Reviewed (Swiss-Prot) only เพื่อใช้ข้อมูลที่ curated แล้ว
  6. กด Search UniProt
  7. ตรวจว่าตาราง UniProt ถูกต้อง โดยดู accession, organism และ length
  8. กด Run AKAP Screen
  9. อ่านผลในตาราง หรือกดดาวน์โหลด CSV เพื่อเก็บผล
รูปที่ 1: ขั้นตอนการใส่ชื่อโปรตีน ค้นหา UniProt และกดรัน AKAP Screen

วิธีแปลผล: ควรดูค่าไหนก่อน?

ให้เริ่มอ่านผลจาก classification ก่อน จากนั้นค่อยดูคะแนนและ warning อื่น ๆ ประกอบกัน

คอลัมน์ความหมายแบบง่ายควรตีความอย่างไร
classificationประเภทของ hitAKAP = น่าสนใจสูง, ambiguous = ยังไม่ชัด, DDIP = อาจจับ d/d-like domain แต่ไม่ใช่ PKA anchor ชัดเจน, unlikely = ความเชื่อมั่นต่ำ
pssm_scoreคะแนนความคล้ายกับ AKAP motif ที่รู้จักยิ่งสูงยิ่งดี แต่ควรดูแยก RI/RII และอย่าใช้ค่านี้อย่างเดียว
ml_probความมั่นใจจาก machine learningค่าใกล้ 1 ดีที่สุด; 0.5–0.7 คือระดับกลาง; ค่าต่ำควรระวัง
n_negdetจำนวน Asp/Glu บน hydrophobic anchor positions0 ดีที่สุด; ถ้า ≥1 ให้ระวัง เพราะอาจรบกวนการจับกับ PKA
predicted_specificityทำนายว่า hit เอนไปทาง RI หรือ RIIเช่น RI-specific, RII-specific, dual, หรือ undetermined
start–end, core, windowตำแหน่งและลำดับกรดอะมิโนใช้สำหรับออกแบบ peptide, mutation หรือการทดลองต่อไป
รูปที่ 2: จุดที่ควรดูในตารางผลลัพธ์ เช่น classification, PSSM score, ML probability และ negative determinants

เลือก hit ไหนไปทำต่อ?

Hit ที่น่าสนใจที่สุดมักมีลักษณะร่วมกันดังนี้

1. classification = AKAP
เป็น candidate ที่โปรแกรมมองว่าเหมือน AKAP มากที่สุด
2. n_negdet = 0
ไม่มี Asp/Glu ไปอยู่บน hydrophobic anchor ที่อาจรบกวนการจับ
3. pssm_score และ ml_prob สูง
ช่วยเพิ่มความมั่นใจว่าลำดับนี้คล้าย AKAP motif
4. specificity ตรงกับคำถาม
เช่น สนใจ RI-specific ก็ควรเลือก hit ที่ทำนายไปทาง RI
รูปที่ 3: หลักการเลือก candidate hit ที่ควร follow-up ด้วยการทดลองหรือการวิเคราะห์โครงสร้าง

ข้อควรระวัง

  • ถ้าใช้ชื่อโปรตีนที่กว้างเกินไป UniProt อาจคืนผลหลาย entry ให้ตรวจ accession ทุกครั้ง
  • ถ้าไม่พบ hit ไม่ได้แปลว่าโปรตีนนั้นไม่มี interaction กับ PKA เสมอไป อาจเป็น isoform อื่น หรือ motif ต่ำกว่า threshold
  • โปรแกรมเป็นเครื่องมือ screening ควร follow-up ด้วย conservation, secondary-structure prediction, AlphaFold, mutation, peptide binding, pull-down, co-IP หรือ localization assay
